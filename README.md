# 记账本 (Expense Tracker)

一个简洁实用的个人记账应用，支持记录收入和支出，提供统计分析功能。

## 功能特性

### 核心功能
- **收支管理**: 添加、编辑、删除收入和支出记录
- **分类系统**: 丰富的支出分类，包含育儿专属分类（奶粉、玩具、童装等）
- **财务仪表盘**: 月度余额概览，收支一目了然
- **交易历史**: 按时间顺序展示所有交易记录
- **统计分析**: 月度和年度支出统计，可视化图表展示

### 特色功能
- **中文界面**: 完全中文化的用户界面
- **育儿模式**: 专属的育儿支出分类，适合有孩子的家庭
- **滚轮日期选择器**: 自定义的滚轮式日期选择器，操作便捷
- **触觉反馈**: 交互时的震动反馈，提升用户体验
- **深色主题**: 现代化的深色配色方案，渐变背景
- **离线存储**: 基于 AsyncStorage 的本地数据持久化

## 技术栈

### 核心技术
- **React Native 0.81.5** - 跨平台移动应用框架
- **React 19.1.0** - UI 库
- **Expo 54** - 开发工具链和 SDK

### 导航与 UI
- **React Navigation 7.x** - 底部标签导航
- **Expo Linear Gradient** - 渐变背景
- **Expo Haptics** - 触觉反馈
- **Ionicons** - 图标库
- **React Native Safe Area Context** - 安全区域处理

### 数据存储
- **AsyncStorage** - 本地数据存储
- **Context API** - 状态管理

## 快速开始

### 环境要求

- Node.js 18+
- npm 或 yarn
- Expo CLI

### 安装依赖

```bash
npm install
```

### 运行应用

```bash
# 启动开发服务器
npx expo start

# 在 iOS 模拟器运行
npm run ios

# 在 Android 模拟器运行
npm run android
```

### 构建应用

项目使用 EAS (Expo Application Services) 进行构建：

```bash
# 构建开发版本
eas build --profile development --platform all

# 构建预览版本
eas build --profile preview --platform all

# 构建生产版本
eas build --profile production --platform all
```

## 需求建议与项目优化点

以下内容可作为下一阶段迭代的需求池与优化清单。

### 一、产品需求（用户价值优先）

1. **预算管理**
   - 支持按月设置总预算与分类预算。
   - 预算超支时在首页和统计页给出醒目提醒。

2. **数据导入导出**
   - 支持 CSV/Excel 导出账单，便于备份和复盘。
   - 支持从历史备份文件导入，降低换机成本。

3. **周期账单**
   - 支持房租、会员、保险等固定账单自动生成。
   - 提供到期提醒，减少漏记。

4. **高级筛选与搜索**
   - 按金额区间、分类、备注关键词、时间范围组合筛选。
   - 支持常用筛选条件保存为“快捷视图”。

5. **多账本与家庭协作（可选）**
   - 支持个人账本、家庭账本、育儿账本独立管理。
   - 通过邀请码共享账本，适配家庭共同记账场景。

### 二、体验优化（交互与可用性）

1. **录入效率提升**
   - 增加“上次金额/分类复用”“一键复制上一笔”能力。
   - 常用分类置顶，并支持自定义排序。

2. **空状态与引导优化**
   - 首次使用提供 3 步上手引导（添加、查看、统计）。
   - 历史页/统计页在无数据时给出示例和行动按钮。

3. **可访问性增强**
   - 适配系统字体放大与屏幕阅读器。
   - 关键操作提供更明显的触觉和视觉反馈。

4. **错误提示友好化**
   - 金额输入、日期选择、存储失败等场景统一错误文案。
   - 提供可恢复操作（重试、撤销、回滚）。

### 三、技术优化（稳定性与可维护性）

1. **状态管理分层**
   - 将业务逻辑从 UI 组件中进一步抽离，沉淀 hooks/service 层。
   - 为核心计算逻辑建立单元测试，降低回归风险。

2. **存储层升级**
   - 中长期可评估从 AsyncStorage 迁移到 SQLite/MMKV。
   - 提升大数据量下查询、筛选、统计性能。

3. **性能优化**
   - 历史列表引入虚拟化优化与分批渲染策略。
   - 针对统计计算增加 memo 缓存，减少重复计算。

4. **工程质量保障**
   - 增加 ESLint + Prettier + Husky 提交前检查。
   - 引入基础 CI（lint/test/build）流水线，确保主干稳定。

5. **错误监控与埋点**
   - 接入崩溃收集与行为埋点（如 Sentry/自定义埋点）。
   - 重点跟踪“添加失败率、次日留存、月活跃记账天数”等指标。

### 四、建议迭代节奏

- **P0（1~2 周）**：预算管理、导出、录入效率优化、错误提示统一。
- **P1（2~4 周）**：周期账单、高级筛选、首屏引导、列表性能优化。
- **P2（4 周+）**：多账本协作、存储层升级、监控埋点与指标看板。
